# 资源占用与性能准则

## 1. 内存管理

- **Rust/Go**: 优先使用流式处理（Streaming）而非一次性将大日志文件读入内存。
- **定时任务**: 确保定时任务（Cron）之间有错峰，避免多个维护脚本同时启动导致 CPU 瞬间满载。

## 2. 二进制文件优化

- **Rust**: 建议在 `Cargo.toml` 中使用 `lto = true` 和 `codegen-units = 1` 以减小体积。
- **Go**: 建议使用 `-ldflags="-s -w"` 编译参数来移除调试信息，压缩体积。
- **目标**: 单个二进制文件大小应尽量控制在 15MB 以内。
